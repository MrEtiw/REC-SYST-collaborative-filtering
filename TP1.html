Il est important de calculer la similitude en cosinus en utilisant la matrice de vote dont on a retir la moyenne de l'utilisateur effectuant chacun des votes et non sur la matrice de votes brutes. Ceci aide mitiger les biais induient par les utilisateurs. Par exemple, on rduit l'effet d'un utilisateur qui voterait en moyenne plus lev qu'un autre utilisateur. En effectuant cette transformation, on tient donc compte des diffrences individuelles entre les utilisateurs.

# On obtient le vecteur contenant les similitudes en cosinus en prenant la ligne correspondant au film star trek
# dans mat_mul et en divisant lment par lment avec le vecteur contenant les normes multiplies par celle de star trek
similitude_cos <-mat_mul[index_star_trek,]/norm_mult

Comme attendu, le film est parfaitement corrl (valeur de 1) avec lui-mme. La distribution est entre -1 et 1 (ce qui est attendu pour des similitudes en cosinus). Puisque les votes sont strictement positifs (entre 1 et 5), une similitude en cosinus calcul sur la matrice brute de votes contiendra des valeurs entre 0 et 1. Par contre, puisque nous effectuons la correction (soustraire la moyenne de l'utilisateur mettant chacun des votes) permettant de mitiger les diffrences individuelles entre les individus, il est possibles d'avoir une ''vote'' ngatif s'il tait en dessous de la moyenne de l'utilisateur. Ainsi, nous obtenons une distribution en similitude de cosinus situe entre -0.25 et 1, ce qui est une distribution attendue pour un cosinus. Mathmatiquement, un cosinus peut varier entre -1 et 1.

    # Tableau de frquence des gens qui aime, spar par sexe
    aime_gender <- table(mx[aime, 'gender'])


    # Tableau de frquence des gens qui n'aime pas, spar par sexe
    aime_pas_gender <- table(mx[aime_pas, 'gender'])


    # Tableau de frquence des gens qui aime, spar par age
    aime_age <- table(mx[aime, 'age'])


    # Tableau de frquence des gens qui n'aime pas, spar par age
    aime_pas_age <- table(mx[aime_pas, 'age'])
    
    # P(E_1 | H)
    P_e1h <- ((aime_job)[profession]+1)/(sum(aime_job)+2)
    if (is.na(P_e1h)){
        P_e1h <- 1/(sum(aime_job)+2)
    }

    
    # P(E_1 | ~H)
    P_e1nh <- ((aime_pas_job)[profession]+1)/(sum(aime_pas_job)+2)
    if (is.na(P_e1h)){
        P_e1h <- 1/(sum(aime_pas_job)+2)
    }

    
    # P(E_2 | H)
    P_e2h <- ((aime_age)[toString(age)]+1)/(sum(aime_age)+2)
    if (is.na(P_e2h)){
        P_e2h <- 1/(sum(aime_age)+2)
    }

    
    # P(E_2 | ~H)
    P_e2nh <- ((aime_pas_age)[toString(age)]+1)/(sum(aime_pas_age)+2)
    if (is.na(P_e2nh)){
        P_e2nh <- 1/(sum(aime_pas_age)+2)
    }

    
    # P(E_3 | H)
    P_e3h <- ((aime_gender)[sexe]+1)/(sum(aime_gender)+2)
    if (is.na(P_e3h)){
        P_e3h <- 1/(sum(aime_gender)+2)
    }


    # P(E_3 | ~H)
    P_e3nh <- ((aime_pas_gender)[sexe]+1)/(sum(aime_pas_gender)+2)
    if (is.na(P_e3h)){
        P_e3nh <- 1/(sum(aime_pas_gender)+2)
    }

    
    # On calcule les odds, en supposant l'indpendance de chacune des vidences!!
    reponse_odds <- O_h * P_e1h/P_e1nh * P_e2h/P_e2nh*P_e3h/P_e3nh
    
    # On convertit la rpons en probabilit
    reponse_prob <- OddsToP (reponse_odds)
    return(reponse_prob)
    
    
    
}
